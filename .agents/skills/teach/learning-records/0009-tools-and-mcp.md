# Tool 与 MCP 概念已建立

用户理解了 Tool 是 agent 的具体能力（每一条 hook 日志记录 = 一次工具调用），以及 MCP 是连接 agent 与外部工具的开放协议。能区分内置 Tool 和 MCP Tool，理解了 MCP 的"USB"类比。

用户项目中尚无 MCP 配置——这提供了下一步动手配置的真实素材。

**Evidence:** 用户能通过 hook 日志识别各种工具调用并说出目的；完成了 Lesson 9 的测验。

**Implications:** MCP 概念已理解，下一步可选择：(1) 动手配置一个真实的 MCP Server，或 (2) 先学习 Workflow——多 agent 编排。
