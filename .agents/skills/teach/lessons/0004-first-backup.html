第一次完整备份Lesson 4 · init → commit → push

前三节课建立了心智模型。这节课是实操——把你当前项目的代码真正备份到 GitHub。

完整流程：6 步

步骤命令做什么
1. 初始化git init在项目根目录创建 .git/，生成一个空仓库
2. 创建 .gitignore（手动写文件）告诉 Git 哪些文件不要追踪（密钥、临时文件、依赖包等）
3. 第一次 commitgit add . + git commit -m "..."把所有项目文件拍进第一张快照
4. 在 GitHub 创建空仓库（浏览器操作）在 GitHub 上创建一个空的远程仓库，获取它的 URL
5. 关联 remotegit remote add origin <URL>告诉本地仓库"远程备份在这里"
6. 推送git push -u origin main把本地的 commit 链发送到 GitHub，完成备份

Step 2 详解：.gitignore

不是所有文件都应该进 Git。你需要一个 .gitignore 文件来排除：

  • 密钥和令牌（.env、*.pem、API keys）—— 泄露到 GitHub 是安全事故
  • 依赖包（node_modules/、venv/）—— 体积大，且可以通过 package.json 等重建
  • 构建产物（dist/、build/、*.pyc）—— 可重新生成
  • 编辑器个人配置（.vscode/、.idea/）—— 团队里每人配置不同

.gitignore 就是一个纯文本文件，每行一个规则。示例：

# 依赖
node_modules/
__pycache__/

# 密钥
.env
*.pem

# 构建产物
dist/
build/

# 操作系统
.DS_Store
Thumbs.db

Step 4 详解：在 GitHub 创建空仓库

  1. 打开 github.com/new
  2. 输入仓库名（比如 my-project）
  3. 不要勾选 "Add a README file"
  4. 不要勾选 "Add .gitignore"
  5. 不要选择 License
  6. 点 "Create repository"
  7. GitHub 会显示一个 URL：https://github.com/你的用户名/my-project.git
为什么不能勾选 README？

如果 GitHub 帮你创建了 README，那远程仓库就有了一个初始 commit。你本地也有一个初始 commit。两个不相关的 commit 链无法直接 push——你得先 pull 再 merge。跳过 README，让远程空着，push 就干净利落。

之后每次备份

第一次设置好后，以后的每一次备份只需要：

git add .
git commit -m "描述你做了什么"
git push

三行，10 秒。AI agent 帮你跑。

"把刚才的改动 commit 并 push 到 GitHub"
