Lesson 7 · 配置你的第一个 Hook

上一篇：Lesson 6 — Agent、Hook、Skill 三块积木

上节课你发现了：本地 settings.local.json 里没有 hook 配置。这节课你将从零配置一个真正可用的 hook——每次 agent 调用工具时自动记录日志。做完你就能亲眼看到 agent "在做什么"。

Hook 的配置结构

Claude Code 的 hook 配置写在 settings.json（或 settings.local.json）的 "hooks" 字段里。结构长这样：

{
  "hooks": {
    "事件名": [
      {
        "matcher": "匹配规则（正则，可选）",
        "command": "要执行的命令"
      }
    ]
  }
}

一个事件可以绑多个 hook（所以用数组），每个 hook 可以加 matcher 只对匹配的工具生效。

可用的事件类型

事件触发时机什么时候用
PreToolUseagent 调用工具之前拦截危险操作、修改工具参数
PostToolUseagent 调用工具之后记录日志、校验输出结果
Stopagent 会话结束时清理临时文件、保存状态
Notificationagent 完成任务通知时桌面弹窗、触发下游流程
SessionStart会话开始时设置环境变量、预热缓存
💡 我们选 PostToolUse

对初学者最友好的事件——不做任何干预，只是"观察" agent 的一举一动。安全、无副作用、可以随时删掉。

你要创建的 Hook

目标：每次 agent 调用任何工具后，自动把工具名和时间追加到一个日志文件。

效果：你对着 Claude Code 说一句话，然后去 ~/claude-tool-log.txt 看看它刚才用了哪些工具。

动手步骤

Step 1 · 打开配置文件

打开 .claude/settings.local.json。它现在大概长这样（有 permissions 和 enabledPlugins）：

{
  "permissions": { "allow": [...] },
  "enabledPlugins": { ... }
}

Step 2 · 添加 hooks 字段

在 "enabledPlugins" 那行的后面加一个逗号，然后加入这段：

"hooks": {
  "PostToolUse": [
    {
      "command": "echo \"[$(date '+%Y-%m-%d %H:%M:%S')] Called: $CLAUDE_TOOL_NAME\" >> ~/claude-tool-log.txt"
    }
  ]
}

完整的 JSON 结构应该是：

{
  "permissions": { "allow": [...] },
  "enabledPlugins": { ... },
  "hooks": {
    "PostToolUse": [
      {
        "command": "echo \"[$(date '+%Y-%m-%d %H:%M:%S')] Called: $CLAUDE_TOOL_NAME\" >> ~/claude-tool-log.txt"
      }
    ]
  }
}

Step 3 · 保存并重启 Claude Code

保存文件后，重启 Claude Code 会话让新配置生效。然后对它说一句：

> 帮我看看今天星期几

Step 4 · 验证：查看日志

执行完后，查看日志文件：

$ cat ~/claude-tool-log.txt

你应该能看到类似这样的输出：

[2026-06-22 14:35:01] Called: Bash [2026-06-22 14:35:02] Called: Read

这就是 agent 的"幕后动作"——你每说一句话，它可能调用了好几个工具。

理解你刚做了什么

这个 hook 虽然只有一行命令，但涉及了几个关键概念：

部分含义
"PostToolUse"监听的事件类型——"工具调用后"触发
"command": "..."要执行的 shell 命令
$CLAUDE_TOOL_NAME环境变量——Claude Code 自动注入，包含被调用的工具名
>> ~/claude-tool-log.txt追加到日志文件（不是覆盖）

Hook 命令可以访问的环境变量包括：$CLAUDE_TOOL_NAME（工具名）、$CLAUDE_TOOL_INPUT（输入参数）等——这意味着你可以记录更丰富的信息。

进阶玩法（可选）

如果你只对特定工具感兴趣，可以加 matcher 字段：

{
  "matcher": "Bash",
  "command": "echo \"⚠️ Bash 被执行: $CLAUDE_TOOL_INPUT\" >> ~/bash-warnings.txt"
}

matcher 支持正则，比如 "Bash|Write|Edit" 匹配多个工具，""（空）匹配全部。

测验：你弄懂了吗？

Q1. 你想在 agent 调用 Write 工具之前拦截并检查文件路径——用哪个事件？

A. PostToolUse
B. PreToolUse
C. Stop

Q2. 你只想监控 Bash 和 Write 两个工具——怎么做？

A. 创建两个 hook
B. 用 matcher 正则
C. A 和 B 都可以

完成检查清单

  • 在 settings.local.json 中添加了 hooks 配置
  • 重启 Claude Code 后，对 agent 说了一句任意的话
  • cat ~/claude-tool-log.txt 能看到工具调用记录
  • 能用自己的话解释 $CLAUDE_TOOL_NAME 是什么
