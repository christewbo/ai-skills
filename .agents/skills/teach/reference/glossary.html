Git 术语表

每个术语附一句话定义 + 用你自己的话解释的空间。

核心概念

术语定义
repository（仓库） Git 的数据库——存在 .git/ 目录下，包含所有 commit 历史和配置。你可以把它想象成一个"快照档案馆"。
commit（提交） 项目在某一个时刻的完整快照，附带作者、时间戳、提交信息和父 commit 的引用。每个 commit 有一个 SHA-1 哈希作为唯一 ID（如 a3f2b91c）。
working directory（工作目录） 你电脑上正在编辑的实际文件夹。Git 监视它，但不自动保存——你需要手动告诉 Git 哪些改动要放进下一个 commit。
staging area / index（暂存区） 一个"购物车"——你挑好哪些文件改动要放进下一张快照。通过 git add 添加，通过 git commit 正式拍进历史。
branch（分支） 一个指向某个 commit 的移动标签。当你做新 commit 时，当前 branch 自动向前移动。默认 branch 通常叫 main。
HEAD 一个特殊的指针，指向"你现在在哪"。通常是某个 branch，也可能直接指向某个 commit（detached HEAD）。
remote（远程） 另一个 Git 仓库的引用，通常在一台远程服务器上（如 GitHub）。用于同步——push 上传，pull/fetch 下载。

基础操作

命令做什么
git init 在当前目录创建一个新的 Git 仓库（生成 .git/ 目录）。
git add <file> 将文件的改动加入暂存区。"我选了这些文件放进下一张照片。"
git commit -m "msg" 将暂存区的内容拍成一张新快照，附上信息。"咔嚓——拍了。"
git status 显示工作目录和暂存区的状态——哪些改了、哪些在暂存区、哪些还没追踪。
git log 查看 commit 历史链。"翻阅以前的照片。"
git diff 显示具体改了什么——工作目录 vs 暂存区，或暂存区 vs 最新 commit。
