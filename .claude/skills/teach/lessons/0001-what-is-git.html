Git 是什么Lesson 1 · 心智模型，不是命令清单

大多数人学 Git 是从命令开始的——git add、git commit、git push。这很自然，但也很危险。命令忘记了可以查，但如果你不知道 Git 在做什么，每次报错都像撞墙。

这节课不讲命令。我们建立 Git 的心智模型——它到底是什么、怎么组织数据、以及为什么它是安全的。这个模型会是你以后理解所有 Git 操作的基础。

Git 是一个快照链

把 Git 想象成一个自动拍照的相机。

每次你按快门，Git 拍下整个项目的完整快照（snapshot）——所有文件，所有文件夹，当时的样子。然后它把这张照片串在一根链条上。下一张照片指向前一张，再下一张指向这张……形成一条不可篡改的历史链。

o --- o --- o --- o
|      |      |      |
快照A   快照B   快照C   快照D
(每个快照包含全部文件的完整副本)

这张照片在 Git 里叫 commit（提交）。一个 commit 包含：

组成部分内容
文件快照那一刻所有被追踪文件的完整内容
作者谁拍的（名字 + 邮箱）
时间戳什么时候拍的
提交信息一段文字说明，解释为什么拍
父指针指向上一个 commit 的链接
关键洞察：Git 存的是完整快照，不是差异

很多版本控制系统存的是"文件 A 改了第 3 行，文件 B 删了第 5 行"这种差异。Git 不这样做——它每次存的是整个项目的完整副本。这听起来浪费空间，但 Git 在背后做了极致压缩（相同内容只存一份）。"存快照"这个设计让 Git 非常快、非常安全——你随时可以回到任何一个历史瞬间。

三个区域：工作区、暂存区、仓库

Git 把你的项目分成三个"区域"：

┌──────────────┐    git add    ┌──────────────┐   git commit   ┌──────────────┐
│              │  ───────────→ │              │ ─────────────→ │              │
│   工作目录    │               │    暂存区     │                │    仓库       │
│ Working Dir  │               │  Staging     │                │  Repository  │
│              │               │              │                │              │
│ 你正在编辑    │               │ 你选好了      │                │ 历史快照链    │
│ 的文件       │               │ 下一张照片    │                │              │
│              │               │ 要拍哪些      │                │              │
└──────────────┘               └──────────────┘                └──────────────┘
      ↑                                                             │
      └─────────────────────────────────────────────────────────────┘
                            git checkout / git restore
                        (从历史中恢复某个版本到工作区)
  • 工作目录（Working Directory）— 你电脑上的实际文件夹。你编辑代码的地方。Git 不自动管这里发生了什么。
  • 暂存区（Staging Area / Index）— 一个"购物车"。你挑好哪些改动要放进下一张照片。这一步是手动的——你决定拍什么。
  • 仓库（Repository / .git/）— 快照链存放的地方。存在项目根目录下一个叫 .git/ 的隐藏文件夹里。这是 Git 的"数据库"。
为什么要有暂存区？

因为没有暂存区，你每次只能把所有改动都拍进一张照片。有了暂存区，你可以把一个大改动拆成多张逻辑清晰的小照片——比如"修复了登录 bug"是一张，"重构了样式"是另一张。这让历史可读，也容易回退。

commit 之间的指针链

每个 commit 都知道自己的"父亲"是谁——这就是父指针。这形成了一条只能往前长的单向链：

A  ←──  B  ←──  C  ←──  D
│       │       │       │
v1.0   加了    修了    加了
       登录页  一个bug   README

每个字母是一个 commit，箭头指向它的父亲。C 知道它父亲是 B，B 知道它父亲是 A。所以你从 D 可以一路回溯到 A——整个项目的历史。

这种结构在数学上叫 DAG（有向无环图，Directed Acyclic Graph）。你不需要记住这个名字，只需要理解：Git 的历史是一条只能往前、不能回头修改的链。一旦拍了照，那张照片就永远在那里。

这也是 Git 安全的根源：

  • 每个 commit 用 SHA-1 哈希作为 ID（像 a3f2b91 这种字符串），内容一旦改变，ID 就变了。
  • 因为每个 commit 里存着父 commit 的 ID，改了历史中的任何一环，整条链的 ID 都会变。
  • Git 能立刻检测到任何篡改或损坏。

branch：给 commit 起个小名

每次输 commit 的哈希 ID（a3f2b91...）很烦。所以 Git 发明了 branch（分支）——就是一个指向某个 commit 的移动标签。

A  ←──  B  ←──  C  ←──  D
                      ↑
                    main
                 (branch 指针)

main 不是一个什么特殊的东西——它就是一个标签，指向 commit D。当你做新的 commit 时，main 自动向前移动到最新的 commit。

你可以创建新 branch（比如 experiment），它最初指向同一个 commit。然后你在新 branch 上继续拍照，两条线就分叉了——这就是"分支"的名字来源。但那是下一课的内容。

总结：四个核心概念

概念一句话解释
commit一张完整快照 + 作者 + 时间 + 说明 + 指向父亲的链接
三个区域工作目录（改文件）→ 暂存区（挑要拍的）→ 仓库（拍进历史）
commit 链每个 commit 指向它的父亲，形成不能篡改的单向历史
branch一个指向某个 commit 的移动标签，方便引用

和你目标的关系

你的目标是通过 AI agent 安全备份代码。现在你应该能看出 Git 为什么是这个场景的最佳工具：

  • 完整快照意味着你随时可以回到任何一个历史状态——删错了文件？git 里有。
  • 不可篡改的链意味着你的历史是安全的——坏了任何一块 Git 都会发现。
  • branch + remote（下节课讲）意味着你可以同步到另一台机器或 GitHub——物理上分离的备份。
