远程仓库与备份Lesson 2 · 让快照离开你的电脑

上一课讲的 commit 链全在你的电脑上，藏在 .git/ 文件夹里。这已经很安全了——但你的电脑坏了呢？被偷了呢？你需要另一份副本放在另一个地方。这就是 remote（远程仓库）的作用。

remote 就是另一个 Git 仓库

技术上，"remote"一点也不特殊——它就是另一个 Git 仓库，只是恰好不在你的电脑上。通常在 GitHub、GitLab 或一台服务器上。

你的电脑 (local)                     GitHub (remote)

A ←── B ←── C ←── D               A ←── B ←── C ←── D
              ↑                                  ↑
             main                                main

             │                                  │
             └──────── git push ────────────────→┘
             ←──────── git pull ────────────────┘
关键洞察：remote 和你本地是平等的

GitHub 上的仓库不是"主"副本。你的本地仓库和 remote 是对等的——各自拥有一套完整历史。这和 Dropbox 不同——Dropbox 是"云端是主，本地是缓存"。Git 是"两边都是完整数据库，通过 push/pull 互相同步"。

三个基本操作

操作方向做什么
git push 本地 → remote 把你本地的新 commit 发送到 remote。"把照片传一份到云端。"
git fetch remote → 本地 下载 remote 上你还没有的 commit，但不改动你的工作目录。"看看云端有什么新东西。"
git pull remote → 本地 = fetch + 自动把你的工作目录更新到最新。"把云端的新东西拿来并应用到我的代码上。"

对备份场景来说，你现在只需要关心 push。fetch 和 pull 是多人协作时用的——先知道它们存在就行。

完整流程：从零到备份

假设你有一个项目目录 my-project/，里面已经有代码。把它变成"安全已备份"状态只需要四步：

Step 1: 在 GitHub 上创建空仓库

打开 GitHub → 点右上角 + → New repository → 起个名字（比如 my-project）→ 不要勾选 "Add a README file" → 点 Create。

GitHub 会显示一段类似这样的指引，其中有一个 URL：

https://github.com/你的用户名/my-project.git

这就是 remote 的地址。

Step 2: 初始化本地 Git

cd my-project
git init

Step 3: 做第一次 commit

git add .
git commit -m "初始版本"

Step 4: 关联 remote 并 push

git remote add origin https://github.com/你的用户名/my-project.git
git push -u origin main

拆解一下这两行：

  • git remote add origin <URL> — 给你的本地仓库添加一个指向远程地址的引用，起个名叫 origin（约定俗成的名字，意思是"来源"）。你本地可以有多个 remote，但通常一个就够了。
  • git push -u origin main — 把你本地的 main 分支推到 origin 上。-u 的意思是"记住这个配对"——今后你只敲 git push 就够了，不用每次指定 remote 和 branch。

之后每次你做了新的 commit，只需要：

git push

就行了。

clone：从零获取一份副本

如果你换了新电脑，或者同事想拿你的代码：

git clone https://github.com/你的用户名/my-project.git

这做了三件事：

  1. 创建一个 my-project/ 文件夹
  2. 下载整个仓库（所有 commit 历史 + 所有文件）
  3. 自动把工作目录设到最新 commit
clone 下来的是什么？

clone 拿到的是完整仓库——整个 commit 链、所有 branch、所有标签，全在你本地。你不依赖 GitHub 也能查看完整历史。GitHub 挂了？你本地有一切。

你的备份工作流

结合你的目标——通过 AI agent 安全备份代码——标准操作是：

1. AI agent 写代码
       ↓
2. git add + git commit  （拍快照，附上说明）
       ↓
3. git push              （推到 GitHub）
       ↓
4. 代码物理上存在于两个地方：你的电脑 + GitHub 的服务器

你的数据现在有三层保护：

  • commit 链的完整性校验——数据损坏能被发现
  • 完整快照历史——可以回到任何时间点
  • 地理冗余——本地和 remote 是两份独立的物理副本
