Branch 实战Lesson 3 · 安全区是怎么工作的

上节课你提了一个关键问题：删 branch 不会删代码，那它怎么起到"安全区"的效果？答案在这节课里——切换 branch 时，Git 会替换你的工作目录。

branch 的三个操作

操作命令做什么
创建分支 git branch <name> 在当前 commit 上贴一个新标签。注意：不切换过去，你还在原来的 branch 上。
切换分支 git switch <name> 把 HEAD 移到那个 branch，同时把工作目录替换成那个 commit 的快照。这就是"安全区生效"的关键。
创建并切换 git switch -c <name> = branch + switch 二合一。最常用的方式。
删除分支 git branch -d <name> 撕掉标签。如果该 branch 上有没合并的 commit，Git 会拒绝删除（保护你）。

完整演示：安全区的运作

跟着每一步在你的终端里跑一遍。每一步后面都有解释。

场景设定

git init demo && cd demo
echo "print('hello')" > main.py
git add . && git commit -m "初始版本"

# 当前状态：
# A (初始版本)
# ↑
# main → A

1. 创建实验 branch

git switch -c experiment

现在 main 和 experiment 都指向同一个 commit A。HEAD 在 experiment 上——下一次 commit 会让 experiment 往前走，main 原地不动。

2. 在 experiment 上写"烂代码"

echo "println!(broken rust in python file?!);;" >> main.py
git add . && git commit -m "实验：乱写一通"

当前状态：

A (初始版本) ←── B (实验：乱写一通)
↑                   ↑
main              experiment (HEAD)

main.py 现在的内容：

print('hello')
println!(broken rust in python file?!);;   ← 烂代码

3. 切回 main——魔法发生

git switch main

再看 main.py：

print('hello')
# 烂代码消失了！

Git 做了三件事：

  1. 把 HEAD 移到 main
  2. 把工作目录里的所有文件替换成 commit A 的快照
  3. 你看到的就是干净版本，就像 experiment 上什么都没发生过

4. 删掉 experiment

git branch -d experiment

B 失去所有标签，变成孤儿 commit。Git 的垃圾回收最终会清理它。

A ←── B (孤儿，待回收)
↑
main (HEAD)

你回到了干净状态，就像那场实验从没发生过。

安全区的完整逻辑

步骤发生了什么
创建 branch给当前 commit 贴一个新标签，工作目录不变
在新 branch 上 commit新快照存在新 branch 的线上，其他 branch 不受影响
切回原 branch工作目录被替换成原 branch 的快照——烂代码从你的文件里消失了
删除 branch标签消失，没合并的 commit 变成孤儿，最终被回收
什么时候 Git 会拒绝切换？

如果你在 experiment 上改了一个文件但还没 commit，直接 git switch main 会失败——Git 不想丢掉你的未保存改动。你有三个选择：

  • git commit 先保存
  • git stash 暂存起来
  • git switch -f main 强制切，放弃改动

这是另一个保护机制——Git 默认不会让你不小心丢代码。

branch 和你的备份目标

在日常使用中，用 branch 的最简模式是：

  1. main 永远保持可工作状态——随时可以 push 到 GitHub 做备份
  2. 每次做新东西，从 main 开一个 branch（git switch -c new-feature）
  3. 在 branch 上 commit、push（备份到 GitHub）——坏了也不怕
  4. 做好了就切回 main 合并；做烂了就删掉 branch，代码回到 main 状态
