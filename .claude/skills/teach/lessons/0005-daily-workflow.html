日常工作流Lesson 5 · 改代码 → 备份的节奏

前四课建立了概念并做了一次热身。这节课练两个日常操作——你以后 90% 的 Git 使用就这两件事。

节奏 1：改代码 → 备份（每次写代码都做）

# 写代码...
git add .
git commit -m "做了什么"
git push

三行，10 秒。commit message 写清楚——三个月后你会感谢自己。

节奏 2：用 branch 安全开发（做新功能时用）

# 1. 从 master 开出安全区
git switch -c new-feature

# 2. 在安全区里随意开发，随时 commit
git add .
git commit -m "新功能：第一步"
# ... 继续写 ...
git add .
git commit -m "新功能：完成"

# 3. 成了 → 合回 master
git switch master
git merge new-feature
git push
git branch -d new-feature

# 3 备选：烂了 → 丢弃（安全区的魔法）
git switch master
git branch -D new-feature

检查理解 ✏️

  1. 你在 new-feature 上做了 3 个 commit，master 上做了 1 个。执行 git merge new-feature 之后，master 的 commit 历史变成几个？
  2. 为什么 git switch -c 比 git branch 更常用？
