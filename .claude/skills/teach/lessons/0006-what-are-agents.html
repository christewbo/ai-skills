Lesson 6 · Agent、Hook、Skill：AI 工具的三块积木

你现在用的 Claude Code、未来要搭建的 agent 系统——它们都建立在三块基本积木之上：Agent（干活的人）、Hook（自动触发器）、Skill（技能包）。理解这三者的区别和关系，是搭建任何 AI 系统的第一步。

1. Agent（智能体）——"干活的人"

一个 agent 是一个能自主执行任务的 AI 程序。你给它一个目标，它自己决定：

  • 用什么工具（读文件？搜代码？调 API？）
  • 按什么顺序做事
  • 遇到问题怎么处理

你现在对 Claude Code 说的每一句话，背后就是一个 agent 在工作。你告诉它"修复这个 bug"，它自己去找文件、读代码、编辑、运行测试——不需要你一步步指挥。

💡 关键区别：Agent ≠ 聊天机器人

聊天机器人只会"回答你这句话"。Agent 会采取行动——操作文件、运行命令、调用工具。它有"手"，不只是"嘴"。

2. Hook（钩子）——"到什么节点就自动做什么"

一个 hook 是在特定事件发生时自动触发的脚本。它不自己干活，它决定"什么时候让什么发生"。

Claude Code 支持多种 hook 事件，比如：

Hook 事件什么时候触发典型用途
PostToolUseagent 调用工具之后校验工具输出、记录日志
Notificationagent 完成一个任务时桌面通知、触发下游流程
Stopagent 会话停止时清理临时文件、保存状态
PreToolUseagent 调用工具之前权限拦截、输入校验
🎯 你可以现在就试试

打开 .claude/settings.json，搜索 "hooks"——你很可能已经有一些 hook 配置。看看它们监听了什么事件，做了什么动作。

3. Skill（技能）——"遇到这种事，按这套流程做"

一个 skill 是一个打包好的指令集，告诉 agent：当用户要做某类事情时，你应该怎么做。

一个 skill 通常包含：

  • 名称和描述——什么时候应该激活这个 skill
  • 步骤指令——激活后 agent 应该遵循的流程
  • 可用工具——这个 skill 可以使用哪些特殊工具

比如你现在用的 /teach 就是一个 skill——它告诉 agent："当用户说 /teach 时，你就是一个老师，读取学习记录、设计课程、写 HTML……"

🎯 你可以现在就试试

打开 .claude/skills/teach/SKILL.md——这就是 teach skill 的指令文件。读一下开头几段，你会看到"你是一个老师"的设定。

三者的关系

把它们放在一起看：

┌─────────────────────────────────────────┐
│              Skill（技能）               │
│  "面对这类任务，按这套流程做"             │
│  ┌───────────────────────────────────┐  │
│  │         Agent（智能体）            │  │
│  │  "我接到任务，我决定怎么做"         │  │
│  │  ┌──────────┐  ┌──────────┐      │  │
│  │  │  Tool 1  │  │  Tool 2  │ ...  │  │
│  │  └──────────┘  └──────────┘      │  │
│  └───────────────────────────────────┘  │
│         ↑                    ↑          │
│     Hook 触发          Hook 触发         │
│  (PreToolUse)        (PostToolUse)       │
└─────────────────────────────────────────┘
积木职责类比
Agent接收目标 → 自主决策 → 执行任务员工
Hook监听事件 → 自动触发脚本闹钟 / 传感器
Skill定义"面对 X 时该怎么做"SOP 手册

简单总结：Skill 告诉 Agent 怎么做，Hook 在特定时刻自动帮 Agent 做事。

你的动手练习

学完这节课，做三件事来巩固：

  1. 找到你本地的 skill 文件——打开 .claude/skills/teach/SKILL.md，读前 30 行，找到"你是一个……"那一段
  2. 找到你本地的 hook 配置——打开 .claude/settings.json，搜索 "hooks"，看看有哪些 hook
  3. 用自己的话回答：你现在用的 /teach 命令，为什么是一个 skill 而不是 hook？
